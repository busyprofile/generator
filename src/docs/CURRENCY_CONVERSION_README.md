# Автоматизация SMS-уведомлений с конвертацией валют

## Описание

Система автоматически обрабатывает SMS-уведомления о переводах и сопоставляет их с ордерами, даже если валюты не совпадают. Конвертация происходит через USDT с допуском ±0.9%.

## Поддерживаемые валюты

- RUB (российский рубль)
- TJS (таджикский сомони)
- KZT (казахский тенге)
- USD (доллар США)
- EUR (евро)
- BYN (белорусский рубль)
- UAH (украинская гривна)
- AZN (азербайджанский манат)
- THB (тайский бат)
- TRY (турецкая лира)
- IDR (индонезийская рупия)
- AED (дирхам ОАЭ)
- KGS (киргизский сом)

## Как это работает

### 1. Парсинг SMS
SMS-сообщения автоматически анализируются для извлечения:
- Суммы перевода
- Валюты (из текста SMS)
- Номера карты (последние 4 цифры)
- Платежного шлюза (по отправителю)

### 2. Поиск ордера
Система ищет подходящий ордер в следующем порядке:
1. **Точное совпадение** - одинаковая валюта и сумма
2. **Конвертация через USDT** - если валюты разные, но суммы эквивалентны с допуском ±0.9%

### 3. Конвертация валют
- Все конвертации происходят через USDT как промежуточную валюту
- Используются актуальные курсы из `services()->market()->getBuyPrice()`
- Алгоритм: `исходная_валюта → USDT → целевая_валюта`

## Настройка допуска

Допуск ±0.9% задан в коде и может быть изменен в:
- `app/Services/Sms/CurrencyConverterService.php`
- `app/Queries/Eloquent/OrderQueriesEloquent.php`

```php
// Изменение допуска на ±1.5%
$converter->isWithinTolerance($order->amount, $smsAmount, 1.5)
```

## Мониторинг

### Метрики для отслеживания
- Процент успешно обработанных SMS
- Количество SMS с конвертацией валют
- Ошибки конвертации валют
- Время обработки SMS

### Алерты
Рекомендуется настроить алерты на:
- Процент успешности < 90%
- Ошибки получения курсов валют
- Большое количество необработанных SMS

## Возможные проблемы

### SMS не распознается
1. Проверить настройки отправителей SMS в платежном шлюзе
2. Убедиться, что формат SMS поддерживается парсером
3. Проверить стоп-слова в базе данных

### Ордер не найден
1. Проверить статус ордера (должен быть PENDING)
2. Убедиться, что пользователь и устройство совпадают
3. Проверить платежный шлюз
4. Проверить допуск конвертации

### Ошибки конвертации
1. Проверить доступность сервиса курсов валют
2. Убедиться, что валюта поддерживается системой
3. Проверить логи на наличие ошибок API
